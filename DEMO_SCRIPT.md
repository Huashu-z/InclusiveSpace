## 场景 1️⃣：🌙 老年人夜间去超市

### 背景与动机
- **用户角色**：62 岁的老年人，计划晚上去老城区的一家超市购物
- **核心问题**：该区域是否安全且方便老年人通行？夜间照明充足吗？有台阶或其他障碍吗？
- **关键指标**：可达性得分、环境因素分析、推荐参数设置

### 开场词
> "假设我们的用户 Linda 是一位 62 岁的退休者，想在汉堡老城区的超市购物。我们来看看 AI 系统如何评估该区域对老年人的可达性。"

### 操作步骤

#### Step 1: 加载演示场景
1. 点击侧栏中的 **"▶ 加载演示场景"** 按钮
2. 选择 **"🌙 场景 1：老年人夜间去超市"**
3. 系统预填充：问题、推荐图层、用户画像（如有必要）

**预期显示**：
- 左上角提示：演示场景已加载
- 问题输入框预填：`"这里是否适合老年人夜间去超市？有充足的照明吗？"`
- 侧栏显示提示：需验证起点、图层、画像已正确设置

#### Step 2: 验证配置
- ✅ **起点**：老城区超市附近 (9.9950, 53.5460) — 已在地图上标记
- ✅ **用户画像**：老年人（Elderly）
- ✅ **选定图层**：台阶、夜间照明、无障碍卫生间、障碍物、坡度

#### Step 3: 发送查询

**预期输出**（AI 响应）：

#### Step 4: 展示关键互动

### 常见面试问题与答案提示

**Q1: 为什么有两种模式（点位 vs 区域）？**
> "用户提问类型不同。如果问'这里适合吗'（具体起点），系统进入点位查询；如果问'哪些地方适合'（没有起点），系统自动进入区域推荐。这样可以满足不同的用户需求，而不必让用户手动选择。"

**Q2: 目前的得分是如何计算的？真实版本会如何改进？**
> "当前使用启发式评估：基于周边缓冲区内的图层统计（台阶数、照明点等）计算得分。真实版本会调用 pgRouting 生成实际的路网可达域（等时线），只统计该可达域内的要素，这样准确度会高得多，能真实反映用户实际能到达的地方。"

**Q3: 成本估算？**
> "当前 mock 模式完全免费，没有 LLM 调用。如果集成真实 embeddings + LLM，预计每次查询成本 $0.02–$0.08（取决于 embedding 模型与 LLM 选择）。我们可以通过缓存热门查询、限制上下文长度、批量计算来降低成本。"

**Q4: 如何防止 hallucination（LLM 编造数据）？**
> "当前 mock 模式基于真实的地图图层数据，不会编造。未来引入 LLM 时，会采用以下策略：
> 1. 严格的系统提示，要求仅引用检索到的内容
> 2. 返回 rawSummary 与 highlights，让用户可以验证来源
> 3. 对敏感信息（无障碍设施位置、成本等）的 fact-checking
> 4. 定期用人工标注的金标准测试集验证准确度"

**Q5: 隐私与合规？**
> "当前地图数据是公开的（地理网数据 + POI），不涉及用户隐私。如果系统收集用户查询日志，会遵循 GDPR（欧盟）和 CCPA（美国）的数据最小化与保留政策。我们不会与第三方共享用户轨迹或敏感查询。"

### 文件结构
```
utils/
  ├── demoScenarios.js          # 演示场景定义 & 加载器

components/
  ├── AgentPanel.jsx            # 更新：演示菜单 + 两种响应格式显示
  ├── Sidebar.jsx               # 传递 startPoint/profile/layers 给 AgentPanel

pages/api/
  └── agent.js                  # 后端路由：智能检测问题类型 + 返回对应模式响应
```

### 核心函数
- `getDemoScenario(scenarioId)` — 获取单个场景配置
- `getDemoScenariosList()` — 获取场景列表（用于菜单）
- `applyScenarioToUI(scenarioId)` — 将场景配置转化为 UI 状态更新
- `generateRegionRecommendations(...)` — 区域推荐响应生成
- `generateMockAgentResponse(...)` — 点位查询响应生成（已有）

### API 路由决策逻辑
```javascript
if (isRegionQuery && !startPoint) {
  // 区域推荐模式
  return generateRegionRecommendations(...)
} else {
  // 点位查询模式
  return generateMockAgentResponse(...)
}
```
